# SEO Publish Workflow

This workflow safely publishes new SEO pages generated by the SEO operator.

The workflow should only be used when the user has explicitly approved publishing changes.

---

STEP 1 — VERIFY FILES

Check the following directory for newly generated pages:

site/src/data/pages/

Confirm that the files exist and are valid JSON.

Ensure that the file names match the intended page routes.

Do not proceed if the files appear incomplete or malformed.

---

STEP 2 — CHECK FOR DUPLICATES

Before publishing, verify that the same page does not already exist.

If a page already exists, do not overwrite it unless explicitly instructed.

---

STEP 3 — VERIFY SEO STRUCTURE

For each generated page confirm that it includes:

- title
- meta description
- URL
- H1
- useful sections
- FAQ section
- internal links
- quote CTA

If any of these elements are missing, stop and report the issue.

---

STEP 4 — PREPARE COMMIT

Add only the relevant SEO page files.

Example command:

git add site/src/data/pages/\*.json

Do not stage unrelated files.

---

STEP 5 — COMMIT CHANGES

Use a clear SEO commit message.

Examples:

SEO: add car insurance manchester page  
SEO: add 10 pet insurance breed pages  
SEO: expand car insurance location cluster

---

STEP 6 — PUSH TO REPOSITORY

Push the commit to the main branch.

git push origin main

This will trigger deployment via Vercel.

---

STEP 7 — POST-PUBLISH CHECK

After publishing, confirm that:

- the pages exist in the repository
- the site builds successfully
- the sitemap includes the new URLs

Then produce a short report summarising:

- pages published
- commit message used
- deployment triggered
